# RISKS & WATCH-OUTS

| Risk | Impact | Mitigation |
|---|---|---|
| **Over-flagging tanks precision** | Detection score craters; reviewer drowns in false positives | Geo-mismatch and foreign `merchant_country` are **traps**. Require *combined* deviations (category **and** country) and per-card baselines, never absolute thresholds. Two pytest trap cases guard this. |
| **UI/detector integration stall** | Two tracks block each other; integration eats the final hours | **Freeze the JSON contract at H0–H2** and build the UI against stub data matching it. Real engine drops in behind the same shape. v2 extends it **additively** (new optional fields only) — see [JSON_CONTRACT.md](JSON_CONTRACT.md). |
| **Graph sparsity** | The signature feature renders nearly empty and underwhelms | Raw device/IP edges are near-zero (device reuse 0, IP reuse 1 pair). The **co_burst edge type is what makes the graph meaningful and must ship** — not optional. |
| **Demo time (7 min)** | Great tool, fumbled pitch, lost points | **Script and rehearse** the demo: queue → reason → ring graph reveals QuickPay → dismiss-and-watch-it-learn → cost slider → simulate-incoming → decision tree → notification. See [DEMO_SCRIPT.md](DEMO_SCRIPT.md). |
| **Tuning overfits the known band** | F1 looks great on the private band, logic is actually brittle | Tune against the band but keep signals grounded in the *pattern* (velocity, baseline deviation), not the band itself. The transaction_id cue never becomes a signal. |
| **Scratch files leak into submission** | Looks sloppy; "engineering craft" is 20 pts | Delete `Downloads\_an*.txt` and `/tmp/*.py` during H18–24 cleanup; verify the repo tree before submitting. |
| **Scope creep into skipped items** | Time bleeds from the 80 pts that matter (detection + reviewer UX) | `IMPLEMENTATION_PLAN.md` lists explicit skips. Hold the line. v2 adds DB/AI/notifications/ingestion **only** as named, bounded modules. |
| **Threshold re-scoring confusion** | Slider feels slow or labels go inconsistent | Slider **never re-scores** — scores are fixed at startup; only the cutoff moves. The cost-slider relabel is **cache-only** (not persisted; scores never change). Keep that invariant. |
| **Atlas latency / availability** | A slow or unreachable cloud DB stalls reads or hard-fails the demo | Reads (`/transactions`) are served from an **in-memory cache loaded once at startup** — no per-keystroke Atlas round-trip. `db.is_connected()` pings once; if `MONGO_URI` is unset or Atlas is unreachable the app falls back to **CSV-backed in-memory** (logged, graceful). The demo **never hard-fails** on the DB. See [ARCHITECTURE.md](ARCHITECTURE.md). |
| **Gemini quota / cost / latency** | API throttling, spend, or slow calls drag the experience or block triage | Calls are **lazy and cached** — `summarize()` is cached per `transaction_id` (persisted as `ai_summary`); `classify()` only runs on the borderline, non-critical branch. With no `GEMINI_API_KEY` the app runs a **full rule-based fallback** (`gemini.*` returns `None`). **AI never gates detection** — it is a reviewer narrative + a tie-breaker only. |
| **Secret management** | A leaked `MONGO_URI` or `GEMINI_API_KEY` exposes the Atlas cluster / AI account | `.env` is **gitignored**; copy `.env.example` → `.env` and **never commit** `MONGO_URI` or `GEMINI_API_KEY`. All secrets are optional and missing ones degrade gracefully; tests run hermetic with **no secrets**. See [API.md](API.md). |
| **Write-through latency on each review** | Every review/undo/ingest round-trips to Atlas, adding per-action lag | **Acceptable** — writes are infrequent (one per analyst decision), not per-keystroke, and update the cache in the same step so reads stay instant. In CSV fallback mode the Mongo write-through is **skipped** entirely (no-op without Mongo). |
